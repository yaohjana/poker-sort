# 🃏 撲克牌排序遊戲（原地排序版）

一個互動式網頁遊戲，幫助學生學習排序原理與演算法概念。此版本強調「原地排序」（in-place sorting），所有操作在同一區域完成，更貼近真實排序演算法的思維。

## 📋 功能特點

- **原地排序操作**：單一卡片區域，無分離的来源/目標
- **選擇交換機制**：點擊一張卡片後再點擊另一張即可交換（classic 的 compare-swap 操作）
- **交換動畫效果**：平滑的 CSS3 動畫，讓交換過程視覺化
- **即時統計**：追蹤比較次數、交換次數、完成度百分比
- **演算法教育**：每種排序方式都附有詳細的演算法概念解釋
- **響應式設計**：支援桌面與行動裝置

## 🎯 教育目標

1. **理解原地排序概念**：所有操作都在同一陣列中進行，不額外使用空間
2. **體驗比較與交換**：透過點擊操作，體會「比較-交換」的步驟
3. **認識排序演算法**：引出 bubble sort、selection sort 等基礎演算法
4. **分析效率**：統計比較次數與交換次數，思考時間複雜度

## 🚀 快速開始

1. 在瀏覽器中打開 `index.html`
2. 選擇排序方式（例如：數字大小）
3. 調整牌數（預設 13 張）
4. 點擊「重新發牌」開始遊戲
5. **點擊一張卡片**（會標記為藍色）
6. **點擊另一張卡片**，兩張會交換位置
7. 重複步驟 5-6 直到所有卡片排好
8. 完成後點擊「檢查答案」

## 🎮 遊戲玩法

### 操作流程

1. **選擇排序方式**：從下拉選單決定要怎麼排（數字、花色、顏色等）
2. **發牌**：點擊「重新發牌」後，系統會隨機產生卡片排列
3. **交換卡片**：
   - 點擊其中一張卡片（會變色、浮起，顯示「已選擇」）
   - 再點擊另一張卡片（兩張瞬間交換位置，有動畫）
   - 每一次點擊都算一次「比較」，每一次成功交換都算一次「交換」
4. **檢查答案**：完成後點擊「檢查答案」驗證是否正確
5. **查看提示**：卡關時可點擊「顯示提示」

### 統計欄說明

| 指標 | 意義 |
|------|------|
| 比較次數 | 每次點擊卡片都算一次比較操作 |
| 交換次數 | 實際完成卡片交換的次數 |
| 完成度 | 目前排列與目標順序吻合的卡片比例（%） |

## 🔧 排序方式一覽

| 選項 | 目標順序 | 教學重點 |
|------|----------|----------|
| 🔢 數字大小 (A→K) | A(1) → 2 → … → K(13) | 基本數值比較 |
| 📈 數字遞增 (2→A) | 2 → 3 → … → K → A | 遞增排序 |
| 📉 數字遞減 (K→2) | K → Q → … → 3 → 2 | 遞減排序、反向思考 |
| 🃄 花色符號 | ♠ → ♥ → ♣ → ♦ | 映射順序、non-numeric 排序 |
| 🎨 花色顏色 | 黑(♠,♣) → 紅(♥,♦) | 分組排序、多重關鍵字 |

## 💡 教學建議

### 第一階段：熟悉操作（5-7 張牌）

- 選擇「數字大小」（最直覺）
- 让学生随意交换，習慣點擊操作
- 解釋什麼是「比較」與「交換」

### 第二階段：引入演算法（8-10 張牌）

- 挑戰「氣泡排序法」：
  - 從第一張開始，與下一張比較，若錯就交換
  - 重複直到全部正確
- 統計交換次數，討論效率

### 第三階段：進階思考（11-13 張牌）

- 練習「選擇排序法」：
  - 每次從未排部分找出最小/最大的，交換到正確位置
- 比較 bubble vs selection 的交換次數差異
- 討論穩定排序的概念（如花色顏色）

### 延伸討論

- **時間複雜度**：O(n²) vs O(n log n)
- **原地排序 vs 額外空間**：為什麼有些排序法需要額外陣列？
- **穩定排序**：如果兩張牌數字相同，花色能否保持原順序？

## 🛠️ 技術實作

- **Pure HTML/CSS/JavaScript**：無框架、無外部依賴
- **CSS3 Animation**：`transform` + `transition` 實現交換動畫
- **DOM 操作**：動態生成卡片，根據 `currentOrder` 陣列渲染
- **事件處理**：點擊事件觸發選擇與交換邏輯
- **狀態管理**：簡單物件 `gameState` 管理遊戲狀態
