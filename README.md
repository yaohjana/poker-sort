# 🃏 撲克牌排序學院

一個互動式網頁遊戲，幫助學生學習排序原理與演算法概念。  
目前支援三種空間模型：

- **原地排序（in-place）**：所有操作在同一列完成
- **額外空間模式（extraspace）**：上方未排序、下方已排序，適合示範選擇／插入排序
- **快速排序模式（quicksort）**：用 `pivot`、左半／右半分區與分治歷程，體驗 Divide & Conquer 的精隨

## 📋 功能特點

- **多種空間模型**：原地排序／額外空間／快速排序「分治桌」
- **選擇交換機制**：點擊一張卡片後再點擊另一張即可交換（classic compare–swap）
- **額外空間模式**：上方「未排序」、下方「已排序 + 虛線插入格」，可示範選擇／插入排序
- **快速排序教學區**：
  - 未分區 → 點一張牌當 `pivot`
  - 依大小放入 `< pivot` 與 `> pivot`
  - 上方保留每一次分區歷程（樹狀、標出左／右／切開）
  - 支援「中間切開，左／右欄同時追蹤」的分治視圖
  - 分治全部完成後，顯示標有「順位 + 金色光暈」的最終排序結果
- **即時統計**：追蹤比較次數、交換次數、完成度百分比
- **正確位置高亮**：可選擇顯示「已在正確位置的牌」，並在卡片下方標示順位
- **多國語言**：目前內建繁中／簡中／英文／日文
- **響應式設計**：支援桌面與行動裝置

## 🎯 教育目標

1. **理解原地排序概念**：在同一陣列中完成所有操作，不額外使用空間
2. **體驗比較與交換**：透過點擊操作，體會「比較–交換」的步驟
3. **認識多種排序演算法**：bubble / selection / insertion / quicksort
4. **觀察效率（不要求複雜度公式）**：透過比較/交換次數，討論哪種步驟比較省力
5. **理解分治與遞迴**：在快速排序模式中，透過樹狀歷程與左／右欄，體會 Divide & Conquer 的流程

## 🚀 快速開始

1. 在瀏覽器中打開 `index.html`
2. 選擇排序方式（例如：數字大小）
3. 調整牌數（預設 13 張）
4. 點擊「重新發牌」開始遊戲
5. **點擊一張卡片**（會標記為藍色）
6. **點擊另一張卡片**，兩張會交換位置
7. 重複步驟 5-6 直到所有卡片排好
8. 完成後點擊「檢查答案」

## 🎮 遊戲玩法

### 共通操作流程

1. **選擇排序目標**：從下拉選單決定要怎麼排（數字、花色、顏色等）
2. **選擇教學預設**：可從「選擇排序／插入排序／氣泡排序／快速排序」快速切換對應空間模式與說明
3. **發牌**：點擊「重新發牌」後，系統會隨機產生一組卡片
4. 依選擇的模式操作（見下）
5. 完成後點擊「檢查答案」，或勾選「顯示正確位置」協助學生自我檢查
6. 卡關時可點擊「顯示提示」

### 原地模式（氣泡排序等）

- 只有 **一排牌**，所有操作都在同一列完成
- 點擊任一張牌使其「被選取」，再點擊另一張牌即可交換
- 適合示範 **bubble sort**、原地 selection sort 等

### 額外空間模式（選擇／插入排序）

- 上方：**未排序區**；下方：**已排序區**
- **選擇排序示意**：從上方選一張牌，直接與下方對應位置交換
- **插入排序示意**：從上方選一張牌，再點下方虛線插入格插入到正確位置

### 快速排序模式

- 上方：**分區歷程樹**，每一列顯示一次 `< pivot | pivot | > pivot` 分區結果，並標示是「左／右／切開」分支
- 中間：目前處理中的 **未分區 + pivot + 左半／右半**，pivot 置中，左右以 pivot 為基準排布
- 下方按鈕：
  - 「對左半部繼續分治」「對右半部繼續分治」
  - 「✂ 中間切開，兩邊同時追蹤」：將左右變成兩欄，各自遞迴處理
  - 每個欄位完成後，會在該欄下方顯示「本區塊排序結果」的小排牌
- 當所有分治都完成時：
  - 顯示「🎉 全部分治完成！」
  - 下方一排顯示 **最終排序結果**，每張牌底下標出 1、2、… 的順位，並以金黃色光暈框起來

### 統計欄說明

| 指標 | 意義 |
|------|------|
| 比較次數 | 每次點擊卡片都算一次比較操作 |
| 交換次數 | 實際完成卡片交換的次數 |
| 完成度 | 目前排列與目標順序吻合的卡片比例（%） |

## 🔧 排序方式一覽

| 選項 | 目標順序 | 教學重點 |
|------|----------|----------|
| 🔢 數字大小 (A→K) | A(1) → 2 → … → K(13) | 基本數值比較 |
| 📈 數字遞增 (2→A) | 2 → 3 → … → K → A | 遞增排序 |
| 📉 數字遞減 (K→2) | K → Q → … → 3 → 2 | 遞減排序、反向思考 |
| 🃄 花色符號 | ♠ → ♥ → ♣ → ♦ | 映射順序、non-numeric 排序 |
| 🎨 花色顏色 | 黑(♠,♣) → 紅(♥,♦) | 分組排序、多重關鍵字 |

## 💡 教學建議

### 初階：演算法三件事 + 遊戲操作（5-7 張牌）

- 固定從「數字大小」開始，先熟悉遊戲的比較與交換/插入節奏
- 用最簡單的話理解演算法：**輸入 → 步驟 → 輸出**（本遊戲輸入是牌的排列，步驟是你照規則做比較/交換/插入，輸出是排好序）
- 讓學生用 `檢查答案` 驗證：你做的步驟有沒有讓輸出正確

### 進階：用不變量思考「每輪結束仍成立」（8-10 張牌）

- 進入額外空間模式，練 `選擇排序` 與 `插入排序`
- 要求學生說出不變量（不用一定用名詞，可用白話）：**每輪結束後，已完成的那一段仍保持有序**
- 做對照討論：同一手牌，用不同策略做，學生比較「思考方式」而不只看結果

### 突破：多步流程寫出來 + 推理（11-13 張牌）

- 引導學生把每輪結束後的牌序寫下來（對應 `exam.html` 的實作題）
- 錯誤就當偵錯：學生要指出「哪一輪的步驟決定」不符合規則
- 完成一次口頭/書面解釋：你怎麼確定插入格（或尾端位置）是正確的

### 延伸討論
- **原地排序 vs 額外空間**：為什麼有些做法會把牌分成已排好/未排好的區域？
- **穩定排序**：如果兩張牌數字相同，花色能否保持原順序？
