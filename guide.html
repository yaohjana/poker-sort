🗺️ 建議教學順序（七步驟）

以下為建議授課流程，可與教案「（五）教學活動」兩節表格併用；鐘點與班級狀況請自行調整。

  1. 演算法生活實驗室（教師引導）：由教師先介紹「輸入 → 步驟 → 輸出」與生活流程例子，再帶領學生進入 algorithm-lab.html。
  2. 拖曳配對通關：學生完成三關拖曳排序／配對練習；全部通關時頁面會跳出通關證明，請學生輸入姓名、截圖後依教師指示上傳至課程 Padlet（教師請自行建立看板並公布連結）。
  3. 遊戲示範（選擇排序與插入排序）：教師於 主遊戲示範兩種演算法各至少一輪：如何選牌、交換或插入、如何檢查；可搭配本頁 操作說明與 演算法小知識。
  4. 學生實作：學生自行操作遊戲（可限制排序法為選擇排序、插入排序），鼓勵口頭說出「這一步為什麼這樣做」。
  5. 分階段測驗：進入 exam.html。「初探」「基礎」階段題型已偏向觀念與選擇，協助程度較弱的學生也能通過前兩階段；後兩階段再拉高綜合與步驟題比例。可依班級狀況調整課堂配時與是否開放提示。
  6. 測驗評量證明：測驗頁提供「評量證明」視窗，可顯示姓名、日期與各階段答對題數；請學生截圖後上傳至 Padlet，作為成績與學習歷程紀錄。
  7. 延伸課程：選修或第二節可進行 循序、迴圈與判斷，銜接程式結構與課綱運算思維。

🎯 設計理念

  • 先以生活實驗室建立「演算法 = 輸入 → 步驟 → 輸出」，再進入撲克牌具象操作，降低抽象門檻。
  • 遊戲承接比較、交換、插入與檢查，與測驗的觀念題互相驗證。
  • 重視能說明「每一步為什麼這樣做」，而不只是點到正確答案。
  • 透過 Padlet 留存截圖，兼顧形成性評量與學習歷程。

🧭 測驗四階段（與網頁對應）

分階段測驗中「初探、基礎、應用、挑戰」與題組分類一致；前兩階段已優先採用選擇題與直觀題，方便學生建立信心。

初探

  • 演算法基本語彙、選擇／插入排序在做什麼。
  • 以觀念釐清為主，題幹與選項較直白。

基礎

  • 插入位置、穩定排序、小規模結果辨識（選擇題形式）。
  • 銜接遊戲中「每一輪」的判斷。

應用

  • 多輪推理、陣列操作與流程整合。
  • 能對照遊戲中的「檢查答案」與統計欄。

挑戰

  • 不變量、可重現性、長步驟書寫（依題目）。
  • 適合程度較佳或第二輪複習。

🧪 評量與 Padlet 建議

  • 實驗室通關截圖：證明完成拖曳三關；建議 Padlet 開一欄「演算法實驗室通關」。
  • 測驗證明截圖：證明分階段答對狀況；建議 Padlet 開一欄「分階段測驗」並註明截止日。
  • 形成性：課堂提問、口語說明、同儕互評可與遊戲操作併行。
  • 隱私提醒：若 Padlet 公開，請依校規決定是否僅顯示座號或暱稱。

🎮 主遊戲操作說明

對應 index.html 主遊戲介面；投影講解時可與實機操作同步。

  • 選擇排序方式：從進階設定選擇排序規則（數字、花色、顏色等）。
  • 發牌：點擊「重新發牌」後，系統會隨機產生卡片排列。
  • 排序：點擊兩張卡片即可交換位置（點擊第一張會標記為選擇狀態）。額外空間模式則依畫面提示選牌與插入。
  • 檢查：完成後點擊「檢查答案」看是否正確。
  • 提示：如果卡住，可以點擊「顯示提示」或打開完成提示勾勾協助思考。
  • 統計：上方統計欄會記錄你的比較次數與交換次數。
  • 重新發牌 vs 重置本局：「重新發牌」會取得新的一組牌；「重置本局」則恢復此次發牌的原有模樣，可重複練習同一組牌。

💡 演算法小學堂（用最簡單方式）

以下依「排序目標」整理說明；與主遊戲內依你選的規則變化的提示一致。

演算法 = 輸入 → 步驟 → 輸出（照規則做就能完成）
  1. 輸入：你手上的牌（目前的排列）。
  2. 步驟：依規則做事（本遊戲包含「比較」以及「交換 / 插入」）。
  3. 輸出：把牌排到正確順序。
  4. 三個重點：每一步說清楚、照做能得到結果、做到某個條件就停止。
你在遊戲裡正在練的，就是演算法步驟
  1. 選擇排序目標（例如：數字大小、遞增/遞減）。
  2. 觀察目前牌的排列，想像「下一步要怎麼做」。
  3. 執行一輪規則：選牌並進行「交換」或在正確位置「插入」。
  4. 按「檢查答案」把結果驗證：你做的步驟有沒有讓輸出正確？

尚未選擇／一般提示

提示：演算法=輸入 → 步驟 → 輸出。先把規則想清楚，再用可照做的步驟完成任務。

建議：先從少量牌開始，感受「比較 → 交換」的節奏。

  1. 在進階設定選擇一種排序目標（例如：數字大小）。
  2. 觀察目前牌的排列，想像「理想順序」應該長什麼樣子。
  3. 點擊一張牌，再點擊另一張牌，完成一次交換。
  4. 重複比較與交換，直到你覺得已經排好為止，再按「檢查答案」。

依數字大小（A→K）

演算法=輸入 → 步驟 → 輸出。本遊戲：輸入是牌的排列；步驟是依規則做「比較」與「交換/插入」；輸出是把牌排成遞增（A→K）。

建議練習模式：氣泡排序思維 —— 一直檢查相鄰兩張是否要交換。

  1. 從最左邊開始，依序看「一對相鄰的兩張牌」。
  2. 如果左邊比右邊大，就交換它們的位置。
  3. 走到最右邊時，最大的一張會被「氣泡」到最後面。
  4. 重複上述步驟，多走幾輪，直到全部都排好。

數字遞增（2→A）

演算法=輸入 → 步驟 → 輸出。本遊戲：輸入是牌的排列；步驟是依規則比較並移動；輸出是遞增排列。

建議練習模式：選擇排序 —— 每回合幫「目前最小」找到正確位置。

  1. 從全部牌中找出目前最小的數字。
  2. 把它換到最左邊的位置。
  3. 接著在剩下的牌中，找出最小的數字，換到第二格。
  4. 持續重複，直到所有位置都被正確的數字填滿。

數字遞減（K→2）

演算法=輸入 → 步驟 → 輸出。本遊戲：輸入是牌的排列；步驟是依規則比較並移動；輸出是遞減排列。

建議練習模式：選擇排序（反向）—— 每回合把「目前最大」換到最前面。

  1. 從全部牌中找出目前最大的數字。
  2. 把它換到最左邊的位置。
  3. 在剩下的牌中繼續找最大值，依序往右排。
  4. 完成後，整排牌會從大到小排列。

依花色符號（♠♥♣♦）

演算法=輸入 → 步驟 → 輸出。本遊戲：輸入是牌的排列；步驟是依花色符號順序做比較與移動；輸出是依指定花色順序排好。

思考：如果每種花色被指定一個數字（權重），排序就跟數字排序一樣了。

  1. 先記住花色順序：♠ > ♥ > ♦ > ♣（黑桃最大、梅花最小）。
  2. 可以把每張牌換算成一個整數權重來想像。
  3. 比較兩張牌時，就只是比較它們的權重大小。
  4. 用跟數字排序一樣的方式完成整體排序。

依花色顏色（黑→紅）

演算法=輸入 → 步驟 → 輸出。本遊戲：輸入是牌的排列；步驟是依花色顏色分組比較與移動；輸出是依指定顏色順序排好。

思考：先按顏色分組，再按數字排序，這就是「兩階段排序」。

  1. 先把所有「黑色」牌（♠,♣）集中到左邊。
  2. 再把所有「紅色」牌（♥,♦）集中到右邊。
  3. 如果想更進階，可以在每一組內再按照數字排序。

📋 常見排序法：使用情境與複雜度

選擇排序（找極值）

使用情境

每輪從未排序區找最小（或最大），與當前位置交換。本程式：原地模式可「選一張與目標位置交換」；額外空間模式可從上方取一張放到下方最後一格。

時間複雜度

  • 最好 O(n)
  • 平均 O(n²)
  • 最壞 O(n²)

空間複雜度

O(1) 原地

插入排序（插對位置）

使用情境

每輪取未排序區一筆，插入已排序區的「正確位置」。本程式：額外空間模式，選上方一張牌，再點下方虛線格插入正確位置。

時間複雜度

  • 最好 O(n)
  • 平均 O(n²)
  • 最壞 O(n²)

空間複雜度

O(1) 原地

氣泡排序（相鄰交換）

使用情境

重複掃描，比較「相鄰」兩筆，順序錯就交換。本程式：原地模式，只交換相鄰的兩張牌，一輪輪把較大者往右推。

時間複雜度

  • 最好 O(n)
  • 平均 O(n²)
  • 最壞 O(n²)

空間複雜度

O(1) 原地

快速排序（分治法）

使用情境

選一筆當 pivot，小於的放左、大於的放右，再對左右兩邊遞迴。本程式：可先理解分治概念；實際分區操作較複雜，以說明與複雜度認識為主。

時間複雜度

  • 最好 O(n log n)
  • 平均 O(n log n)
  • 最壞 O(n²)

空間複雜度

O(log n) 遞迴堆疊
